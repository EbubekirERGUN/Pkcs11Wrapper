# Release note draft - v0.1.0

## Summary

`Pkcs11Wrapper v0.1.0` is the first versioned public milestone for the repository.

It establishes a production-minded .NET 10 PKCS#11 wrapper baseline with Linux and Windows validation work, PKCS#11 v3 interface/message API coverage, release verification discipline, and an in-repo Blazor Server admin panel initiative.

## Highlights

- Modern .NET 10 PKCS#11 wrapper with NativeAOT-aware interop design
- Configurable initialization flow with `CK_C_INITIALIZE_ARGS` and operating-system locking support
- PKCS#11 v3 interface discovery support (`C_GetInterface`, `C_GetInterfaceList`)
- PKCS#11 v3 message API support when exposed by the module
- `C_LoginUser` and `C_SessionCancel` managed surface coverage
- Strong Linux validation with fixture-backed SoftHSM regression + NativeAOT smoke
- Real Windows runtime regression path using SoftHSM-for-Windows + OpenSC
- Early Blazor Server admin panel foundation for devices, slots, keys/objects, sessions, and audit visibility

## Notes

- Runtime behavior still depends on the target PKCS#11 module, token, and vendor policy.
- Linux currently remains the deepest validation environment because it also carries the NativeAOT smoke path.
- The admin panel is already functional but is still evolving toward stronger production hardening and richer operations UX.

`Pkcs11Wrapper v0.1.0`
